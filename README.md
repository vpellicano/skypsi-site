# skypsi-web - generated SkyPSI website deployment

This public repository is the generated deployment mirror for the SkyPSI
website. GitHub Pages publishes it at **https://skypsi.com**.

The private source of truth is
**`vic-skypsi/skypsi-marketing`**, under `web/` and `brand/`. Do not author
website source, copy, or brand assets in this repository.

## How an update goes live

1. Edit and verify source in `vic-skypsi/skypsi-marketing`.
2. Generate `index.html` from `web/site-src/` in that repository.
3. Commit only the generated deployment output here and push `main`.
4. GitHub Pages rebuilds and publishes automatically, usually within a minute.

That push is the entire deploy. There is no separate build or release step.

## For an AI agent updating this site

- The homepage is generated as `index.html` at the repo root. Do not hand-edit it.
- Keep the **`CNAME`** file (contents: `skypsi.com`). It maps the custom domain; deleting it breaks the site.
- Keep **`.nojekyll`** (disables GitHub's Jekyll step so files or folders with a leading underscore are served as-is).
- The current page is the STEALTH GATE: wordmark + password field, content
  shipped AES-encrypted. It is generated by
  `web/site-src/build_gate.py` in the private
  `vic-skypsi/skypsi-marketing` repository.
- HTTPS is automatic (GitHub provisions and renews the certificate). Don't disable it.
- DNS is already set (apex points to GitHub Pages, www redirects to the apex). You do not need to touch DNS to change the page.

## Deployment contract

- Keep only static, publishable output here.
- If the private source adopts a site generator, the built static output must
  land on the branch and path GitHub Pages serves. Keep `CNAME` and
  `.nojekyll` in the published output.

## Brand assets

Colors, type, logos, imagery, and voice live in the private
**`vic-skypsi/skypsi-marketing`** repository under `brand/`. Pull them from
there during the build so this deployment mirror cannot become a second source.

## Local preview

It's static. Open `index.html` directly, or run `python3 -m http.server` in the repo and open the local URL.
